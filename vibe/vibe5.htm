● 산악회 커뮤니티

로그인(이전 login-join-builder Skill 사용)과 글쓰기가 가능한 산악회 커뮤니티 페이지를 작성하여 데이터베이스와 웹 사이트를 배포한다.


[c:]-[data]-[cursor] 폴더에서 hiking-community 폴더를 생성하여 데이터베이스 관련 프로그램들을 작성한다.


1. 메인(홈) 페이지 작성

[https://getdesign.md/]-[Airbnb] DESIGN.md 내용을 복사하여 DESIGN.md 파일을 생성해 붙여 넣는다.

목표: 아래 기술스택과  내용을 참고하여 '산들바람' 산악회 커뮤니티 메인 페이지를 작성한다.

[기술스택]

1. 웹 서버: Express + EJS

2. 데이터베이스: SQLite + Prisma ORM (데이터베이스 이름: hiking.db)

3. 디자인: 'Tailwind' CDN, 'lucide' CDN

4. 서버 구동: `npm run dev`

[메인페이지 레이아웃]

1. 모바일과 데스크톱 화면을 모두 지원하는 반응형 페이지로 작성한다.

2. 네비게이션 바(Navigation Bar):

- 좌측에  산 모양 아이콘과 산악회 이름('산들바람'), '산행 일정', '활동 후기', '자유게시판'

- 우측에 '로그인' 표시

- 각 페이지는 나중에 제작예정

3. 히어로 섹션(Hero Section):

- 배경 이미지: 산악회와 어울리는 배경이미지를 검색하여 링크

- 메인 카피: '혼자 가면 지치지만, 함께 가면 정상입니다.'

- 서브 카피: '초보 등린이부터 전문 백패커까지, 매주 대한민국 방방곡곡 멋진 명산으로 떠나는 즐거운 등산 커뮤니티'

- CTA(행동 유도) 버튼: '이번 주 산행 일정 보기'

4. 메인 콘텐츠(Main Contents): 1행 4열 그리드 레이아웃에 활동 후기 4장의 사진과 제목, 간단한 내용을 카드로 출력

5. 푸터(Footer): 간단하게 더미 데이터로 생성

[디자인 요구사항]

1. @DESIGN.md (Airbnb) 디자인 마크다운 파일을 참고하여 테마를 설정한다.

2. Tailwind CSS를 조합해서 'shadcn/ui' 디자인이랑 똑같이 생긴 '버튼', '카드상자'로 깔끔하게 교체한다.

3. 카드(박스)에 마우스를 올리면 살짝 떠오르는 부드러운 애니메이션 효과와 고급스러운 그림자를 추가한다.

4. 아이콘은 'lucide  라이브러리를 사용해 홈페이지에 어울리는 그래픽 요소들로 배치한다.

3. 자유 게시판 페이지 작성

/create-skill

1. Skill name: posts-skill-builder

2. Description: 다양한 프레임워크에 맞춰 게시글 목록, 등록, 정보, 수정, 삭제(CURD) 작업을 수행하기 위한 모든 프로젝트에서 사용할 개인 스킬이다.

3. Trigger Keywords: 게시글, 게시판, Post, board

4. Skill Prompt (AI 행동 지침)

[데이터베이스 요구사항]

1. Prisma와 SQLite을 연동하여 현재 데이터베이스에 'Posts' 테이블 스키마를 정의한다.

2, 테이블 필드: id, writer(작성자:nickname), title(제목), content(내용) createdAt(작성일), updatedAt(수정일)

[게시글 목록 페이지 기능]

1. 최근에 작성된 글 순으로 데스크톱인 경우 표(id, title, writer, createdAt)로, 모바일 경우 카드로 출력한다.

2. 검색어 입력창과 검색 버튼을 왼쪽 상단에 배치하고 검색기능(제목, 내용, 작성자)을 구현한다.

3. 페이지네이션은 데스크톱인 경우 (itemsPerPage:5, visiblePages:10), 모바일인 경우 간단한 UI(이전, 다음)를 사용한다.

4. 게시글 제목을 클릭하면 게시글 정보 페이지로 이동한다.

5. 게시글 목록 오른쪽 상단에 '글쓰기' 버튼을 배치한다.

- 로그인 전: 로그인 페이지로 이동 후 로그인에 성공하면  게시글 쓰기 페이지로 이동

- 로그인 후: 게시글 쓰기 페이지로 이동

[게시글 쓰기 페이지 기능]

1. 제목, 내용을 입력하여 저장한 후 게시글 목록 페이지로 이동한다.

2. 제목, 내용을 입력하기 전에는 '취소', '등록' 버튼 상태를 Disabled 상태로 유지한다.

[게시글 정보 페이지 기능]

1. 해당 게시판 글 정보를 카드(Card) 레이아웃 형태로 출력한다.

2. 게시글 정보 카드(Card) 내가 쓴 글인 경우에만 오른쪽 상단에 '수정', '삭제' 버튼을 배치한다.

3. 삭제 시 '정말로 게시글을 삭제하실래요?' 모달창을 띄우고 '예'인 경우에만 삭제 후 게시글 목록 페이지로 이동한다.

[게시글 수정 페이지 기능]

1. 수정 내용 저장 시 '정말로 게시글을 수정하실래요?' 모달창을 띄우고 '예'인 경우에만 저장 후 게시글 정보 페이지로 이동한다.

2. 제목, 내용을 수정하기 전에는 '취소', '등록' 버튼 상태를 Disabled 상태로 유지한다.

[UI/UX 요구사항]

1. 모바일과 데스크톱 화면을 모두 지원하는 반응형 페이지로 작성한다.

2. 게시글 내용 입력은 `Quil` 에디터를 사용한다.

3. 작성일, 수정일 출력은 'YYYY-MM-DD HH:mm:SS' 포맷으로 출력한다.

4. 모든 모달창의 헤더는 기본 버튼의 테마 색상과 같도록 설정한다.

[요구사항]

1. 게시글 프로그램을 생성하여 네비게이션 바 '자유 게시판' 메뉴와 연결한다.

2. 게시글 관련 모든 페이지는 메인 콘텐츠 넓이와 같게 디자인한다.

3. 게시글 등록은 로그인한 사용자만 가능하도록 한다.

4. `Posts` 스키마에 `writer` '초록이'와 '등린이', 제목과 내용을 더미 데이터로 37개를 등록해줘.

6. SQLite 데이터베이스를 Supabase로 마이그레이션(Migration)

수파베이스는 백엔드 개발 없이 데이터베이스, 로그인, 파일 저장 등을 API 형태로 바로 쓸 수 있게 해주는 오픈소스 백엔드 서비스이다.

● 수파베이스에 프로젝트 생성

1) https://supabase.com/ 사이트에서 회원가입(Sign Up)을 하고 로그인(Sign In)한다.

2) [Your Organizations/Start your project]-[New Project]에서 프로젝트 이름(hiking)과 비밀번호(자동생성 후 저장)를 입력한다.

3) 비밀번호를 분실한 경우 [hiking 프로젝트]-[Connect]-[Direct]-[Connect your app]-[Connection string]-[Reset password]로 새로 생성한다.

● MCP를 사용한 슈파베이스와 커서 연결

MCP는 모델 콘텍스트 프로토콜(Model Context Protocol)의 약자이며, AI가 개발자의 컴퓨터 환경(컴파일러, 데이터베이스, 파일 시스템 등) 및 외부 도구와 직접 소통하고 행동할 수 있도록 연결해 주는 표준 통신 규격(브리지)을 뜻한다.

1) [Supabase 로그인]-[프로필 아이콘]-[Account]-[Access Tokens]-[Generate new token] 버튼을 클릭한다.

2) 토큰이름(Name)에 'cursor-mcp'를 입력하고 토큰 문자열을 반드시 따로 복사해 둔다. (창을 닫으면 다시 볼 수 없다)

3) Cursor 프로젝트 루트 디렉터리에 .curor 폴더를 생성하고 내부에 mcp.json 파일을 만들고 아래 코드를 그대로 붙여 넣는다.

{

  "mcpServers": {

    "supabase": {

      "command": "npx",

      "args": [

        "-y", "@supabase/mcp-server-supabase@latest",

        "--access-token", "여기에 1단계에서 복사한 액세스 토큰 입력"

      ]

    }

  }

}

4) [Cursor]-[Settings]-[Tools & MCP] 활성화하면 최초 활성화 시 Supabase 브라우저 창이 열리고 권한승인(Authorize)을 하면 연결이 완료된다.

● MCP를 사용한 데이터베이스 마이그레이션

로컬 SQLite 데이터베이스를 클라우드 Supabase의 'hiking' 데이터베이스로 마이그레이션 해줘.

1) [Supabase 사이트]-[hiking 데이터베이스]-[Connect]-[ORM]-[Configure ORM]에서 DATABASE_URL를 복사한다.

2) 복사한 URL을 .env 파일에 등록 후 [YOUR-PASSWORD] 자리에 데이터베이스 비밀번호(xxxx)를 입력한다.

내 소스코드를 로컬 SQLite 대신 클라우드 Supabase의 데이터베이스를 사용하도록 수정해줘.
